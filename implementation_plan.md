# NodeManager 开发实施计划

NodeManager 是一个个人订阅管理助手，旨在帮助用户管理服务续费周期并通过 Telegram 提醒。

## 1. 环境准备
- [x] 克隆 GitHub 仓库。
- [x] 安装 Node.js 依赖 (`npm install`)。
- [ ] 创建 `.env` 配置文件（基于 README）。
- [ ] 验证项目是否能在本地运行 (`npm run dev`)。

## 2. 现有功能审计
- [ ] 检查 `data/db.json` 的读写逻辑。
- [ ] 验证 `cronJob.js` 的定时任务是否正确。
- [ ] 测试 `telegramService.js` 的通知功能（需配置机器人）。
- [ ] UI/UX 体验检查（特别是响应式和暗色模式）。

## 3. 待开发/优化功能清单
- [ ] **多用户系统**：目前支持单管理员，考虑增加多用户隔离。
- [ ] **数据导入导出**：支持 CSV 或 JSON 格式的订阅数据批量操作。
- [ ] **增强图表分析**：增加年度/月度费用预测图表。
- [ ] **多通知渠道**：集成 Server酱 (微信)、Bark (iOS) 等。
- [ ] **增强安全性**：完善 JWT 校验逻辑。

## 4. 后续任务安排
1. 运行项目并截图/检查当前 UI 状态。
2. 修复可能存在的遗留 Bug。
3. 按照优先级实现新功能。
